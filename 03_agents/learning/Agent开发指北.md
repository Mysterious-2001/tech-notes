# 什么是Agent
Agent本质就是利用大模型进行感知-决策-执行-反馈，然后循环的系统，上下文压缩，Skill，MCP,RAG，子agent等能力的新增都应该在循环的外部，而不改变循环本身。
![[Pasted image 20260330153801.png|471]]
且新能力基本只通过三种方式接入：扩展工具集和 handler、调整系统提示结构、把状态外化到文件或数据库，不应该让循环体本身变成一个巨大的状态机，模型负责推理，外部系统负责状态和边界。

# Agent和Workflow的区别
核心区别是执行链路是预定义的还是由模型决定的。执行路径由代码预先写死的是 Workflow，由 LLM 动态决定下一步的是 Agent，核心区别在于控制权掌握在谁手里，现实中很多标着 Agent 的产品，深入看其实更接近 Workflow，不过两者本身并无高下之分，真正重要的是给任务找到更适合的解决方案。
