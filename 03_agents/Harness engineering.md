Harness的三根支柱：


| 支柱        | 解决什么问题          | 核心手段                                    |
| --------- | --------------- | --------------------------------------- |
| **上下文工程** | Agent 怎么知道该做什么  | 渐进式文档导航、仓库即唯一真相源、执行计划版本化                |
| **架构约束**  | Agent 怎么不出错     | 确定性 Linter 和 Hook、结构测试、分层架构规则、可观测性与验收体系 |
| **垃圾回收**  | Harness 自身怎么不腐化 | 技术债自动检测、文档新鲜度维护、周期性扫描                   |


核心等式：**更严格的运行时约束 = 更多的 AI 自治能力 = 更大的效率提升**。初看反直觉，但逻辑很直接：约束消除了歧义和试错，Agent 不再需要猜"这里该用什么风格"、"改完该跑什么命令"，而是从环境中直接获得确定性答案。
